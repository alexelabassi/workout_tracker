Anexa 4 – Contractul API (selecție)
Toate endpoint-urile, cu excepția autentificării și a verificării de sănătate, cer un token JWT
valid. Erorile folosesc un format uniform, cu un cod stabil, de exemplu TEMPLATE_NOT_
FOUND, ACTIVE_WORKOUT_EXISTS sau SEARCH_UNAVAILABLE.
Autentificare : POST /api/auth/register
POST /api/auth/login
POST /api/auth/refresh
POST /api/auth/logout
GET /api/auth/me
Planificare : GET /api/exercises
GET|POST /api/routines
GET|POST /api/gyms
GET|POST /api/gyms/{id}/equipment
PUT|DELETE /api/equipment/{id}
Programe : GET|POST /api/templates
GET|PUT|DELETE /api/templates/{id}
POST /api/templates/{id}/publish
POST /api/templates/{id}/unpublish
Sesiuni : POST /api/workouts/start
GET /api/workouts/active
POST /api/workouts/{id}/finish
POST /api/workouts/{id}/cancel
POST /api/session−exercises/{id}/sets
PUT|DELETE /api/sets/{id}
Istoric : GET /api/history
GET /api/analytics/overview
Marketplace : GET /api/marketplace/templates
POST /api/marketplace/templates/{id}/vote
POST /api/marketplace/templates/{id}/save
POST /api/marketplace/templates/{id}/use
Analizator : GET /api/templates/{id}/analysis
Cautare : GET /api/search/templates
GET /api/search/workouts
POST /api/admin/search/reindex
Coaching : GET /api/coach/clients/{id}/history
GET /api/coach/clients/{id}/analytics
GET /api/coach/clients/{id}/sessions
