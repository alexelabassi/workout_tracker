\begin{abstract}{romanian}
Lucrarea prezintă o platformă web pentru planificarea și monitorizarea antrenamentelor de forță,
construită cu accent pe corectitudinea datelor și pe controlul accesului. Aplicația permite unui
utilizator să își gestioneze exercițiile, rutinele, sălile și echipamentele, să construiască programe
pe mai multe zile, să execute sesiuni live, să consulte istoricul și progresul, să publice sau să
copieze programe dintr-un marketplace și să colaboreze, în mod controlat, cu un antrenor.

Problema centrală tratată este separarea dintre datele de planificare, care se pot modifica, și
istoricul antrenamentelor, care trebuie să rămână stabil. La pornirea unei sesiuni, sistemul copiază
înregistrările relevante din program, sală, exerciții, grupe musculare, rutine și echipamente în
instantanee istorice. Astfel, o sesiune finalizată rămâne corectă chiar dacă ulterior utilizatorul
editează sau șterge datele sursă. Pe lângă acest model de date, lucrarea descrie autentificarea cu
JWT și rotația token-urilor de reîmprospătare, protecția față de accesul neautorizat la resurse
(IDOR), autorizarea pe roluri și pe relații pentru modul de coaching, un analizor determinist al
structurii programelor și un model de citire separat pentru căutare full-text cu OpenSearch.

Backend-ul este implementat în Spring Boot 3 / Java 21, cu PostgreSQL 16 ca sursă unică de adevăr și
schema gestionată exclusiv prin Flyway. Frontend-ul este o aplicație React cu TypeScript. Căutarea
este evaluată experimental printr-o comparație între OpenSearch și o variantă PostgreSQL construită
corect, pe un set de date generat până la 2 milioane de documente. Rezultatele arată că PostgreSQL
este suficient pentru colecții mici, dar că OpenSearch devine justificat la dimensiuni mai mari, mai
ales pentru căutarea tolerantă la greșeli de tastare.
\end{abstract}

\begin{abstract}{english}
This thesis presents a web platform for planning and tracking strength training, built with a strong
focus on data correctness and access control. The application lets users manage exercises, routines,
gyms and equipment, build multi-day training programs, run live workout sessions, inspect history and
progress, publish or copy programs through a marketplace, and collaborate with a coach under explicit
relationship-based access rules.

The central problem is the separation between editable planning data and historical workout data that
must remain stable. When a workout session starts, the system copies the relevant program, gym,
exercise, muscle-group, routine and equipment data into historical snapshots. As a result, a completed
session remains correct even if the source data is later edited or deleted. Besides this data model,
the thesis describes JWT authentication with refresh-token rotation, IDOR-resistant resource access,
role-based and relationship-based authorization for coaching, a deterministic program-structure
analyzer, and a separate OpenSearch read model for full-text search.

The backend is implemented in Spring Boot 3 / Java 21, with PostgreSQL 16 as the single source of
truth and Flyway as the only schema-management mechanism. The frontend is a React and TypeScript
single-page application. Search is evaluated empirically by comparing OpenSearch with a properly
indexed PostgreSQL baseline over a generated corpus scaled up to 2 million documents. The results
show that PostgreSQL is enough for small collections, while OpenSearch becomes justified at larger
sizes, especially for typo-tolerant search.
\end{abstract}
