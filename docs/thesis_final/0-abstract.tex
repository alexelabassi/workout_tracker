\begin{abstract}{romanian}
Lucrarea prezintă proiectarea și implementarea unei platforme full-stack pentru antrenamente de
forță, construită pe principiul \emph{adevărului istoric}: planurile de antrenament se transformă,
la execuție, în istoric imuabil prin \emph{instantanee} (snapshots), astfel încât editarea sau
ștergerea ulterioară a datelor sursă nu poate modifica ce a fost înregistrat. Sistemul oferă
gestiunea programelor de antrenament, execuția sesiunilor live, analize de progres, un marketplace
de programe (publicare, vot, salvare, copiere), o analiză structurală deterministă și explicabilă a
programelor (bazată pe euristici informate de dovezi, fără pretenții medicale), un mod de
\emph{coaching} cu acces controlat la datele clientului și o componentă de \emph{căutare} full-text
peste programe și istoricul de antrenament. Componenta de căutare este un model de citire derivat,
construit peste OpenSearch, menținut consistent prin evenimente după-commit, cu securitate în două
straturi. Contribuția centrală este o \emph{evaluare empirică} OpenSearch vs.\ PostgreSQL pe un
corpus sintetic scalat până la 2 milioane de documente. Backend-ul este Spring Boot~3 / Java~21 cu
PostgreSQL~16 ca sursă unică de adevăr, iar corectitudinea este verificată prin teste de integrare
pe baze de date reale (Testcontainers, fără mock-uri). Rezultatele arată un punct de echilibru în
jurul a 10.000 de documente și un avantaj decisiv al motorului de căutare la căutarea tolerantă la
greșeli de tastare.
\end{abstract}

\begin{abstract}{english}
This thesis presents the design and implementation of a full-stack strength-training platform built
on the principle of \emph{historical truth}: when a workout is executed, the plan is frozen into
immutable \emph{snapshots}, so later edits or deletions of the source data can never alter what was
recorded. The system provides workout-program management, live session execution, progress
analytics, a program marketplace (publish, vote, save, copy), a deterministic and explainable
structural analysis of programs (evidence-informed heuristics, no medical claims), a \emph{coaching}
mode with controlled access to a client's data, and a full-text \emph{search} component over
programs and training history. The search component is a derived read model built on OpenSearch,
kept consistent through after-commit events, with two-layer security. The central contribution is an
\emph{empirical evaluation} of OpenSearch versus PostgreSQL on a synthetic corpus scaled up to two
million documents. The backend is Spring Boot~3 / Java~21 with PostgreSQL~16 as the single source of
truth, and correctness is verified through integration tests on real databases (Testcontainers, no
mocks). The results show a crossover point around ten thousand documents and a decisive advantage
for the search engine on typo-tolerant (fuzzy) queries.
\end{abstract}
